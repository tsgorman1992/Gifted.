gifted. Email Drip — Final Implementation Brief with Schema Confirmed
Platform context
gifted. (gifted.page) — premium SMS-driven digital gifting platform. Email service is Resend. All trigger logic lives in our custom scheduler (runs every 10 minutes) or explicit API calls in route handlers. No native Resend automations.

Schema changes — do these first
gifts table:
Add abandonedNudgeSentAt timestamp, nullable.
users table:
Extend existing dripStep state machine from 0-2 to 0-3.
Current:

0 = no drip sent
1 = Email 1 sent
2 = Email 2 sent

Add:

3 = Email 3 sent

New table — email_logs:
sqlCREATE TABLE email_logs (
  id              SERIAL PRIMARY KEY,
  userId          INTEGER REFERENCES users(id),
  email           TEXT NOT NULL,
  type            TEXT NOT NULL,
  sentAt          TIMESTAMP NOT NULL DEFAULT NOW(),
  resendMessageId TEXT,
  status          TEXT DEFAULT 'sent'
);
Type values: drip1, drip2, drip3, digest, abandoned_nudge
Write a row to email_logs after every successful Resend API call across all email functions — existing and new. If Resend returns a messageId, store it in resendMessageId.

Pre-launch requirement for all emails
Add UTM parameters to every CTA button before any email goes live:

utm_source=email
utm_medium=drip
utm_campaign=recipient_to_sender
utm_content=drip1 (adjust per email type)

For abandoned nudge: utm_campaign=abandoned_gift

Task 2 — Email 1 rewrite with personalization
Trigger: scheduler checks daily for users where:

Account createdAt = 3 days ago
recipientUserId exists on at least one gift record
dripStep = 0
hasSentGift = false

Personalization query:
sqlSELECT senderName, occasion, experience, amount
FROM gifts
WHERE recipientUserId = user.id
ORDER BY createdAt ASC
LIMIT 1

Task 4 — Email 3 utility email
Trigger: scheduler checks daily for users where:

dripStep = 2
dripLastSentAt < now - 30 days
hasSentGift = false

Two versions based on occasions data:
Version A — user has upcoming occasions:
Query:
sqlSELECT c.name, co.label, co.month, co.day, co.floatingKey
FROM contactOccasions co
JOIN contacts c ON co.contactId = c.id
WHERE co.userId = user.id
Run through existing computeFloatingDate logic. Filter to ≤ 60 days from today. Sort ascending. Cap at 5.
Subject: A few dates worth knowing about.
Body: Clean table — contact name, occasion label, days until. One "Build a moment →" button per row. No emotional copy. Pure utility.
Version B — no upcoming occasions:
Subject: You'll thank yourself for this in about 3 weeks.
Body:
Add one person and their birthday.

We'll remind you before it arrives — 3 days before
and the morning of. You'll never send a lazy
last-minute gift again.

[Add a birthday →]
CTA links to contacts/people section of dashboard.
On successful send either version:

Set dripStep = 3 on user record
Update dripLastSentAt to now
Write row to email_logs with type = 'drip3'
